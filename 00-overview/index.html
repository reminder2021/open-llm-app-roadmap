LLM 应用开发路线
主页总览

总览与学习方式

这条路线面向“基于现有开源模型做应用”的目标，强调工程闭环：模型调用、知识接入、工具执行、微调、评估和部署。

建议周期：第 1 周产出：个人学习环境与项目仓库

阶段导航

00 总览 01 基础 02 开源模型 03 Prompt 04 RAG 05 工具与框架 06 微调 07 评估 08 部署 09 项目

你要成为哪类开发者

你的目标不是设计 Transformer 或训练基础模型，而是把开源模型变成可用系统。核心能力是：理解模型边界、接入业务数据、设计工具调用流程、用微调改善特定任务，并让系统稳定运行。

主线建议：基础 API → 开源模型推理 → Prompt/结构化输出 → RAG → 工具调用 → LangChain/LangGraph → 微调 → 评估 → 部署。

推荐技术栈

  • 语言：Python 为主，少量 JavaScript/TypeScript 用于前端。
  • 后端：FastAPI、Pydantic、Uvicorn。
  • 模型服务：Ollama、vLLM、Transformers、llama.cpp。
  • RAG：FAISS、Qdrant、Milvus 或 Chroma；Embedding 与 Reranker 选择中文友好的模型。
  • 编排：先手写核心逻辑，再学 LangChain，复杂 Agent 重点看 LangGraph。
  • 微调：Transformers、PEFT、TRL、LLaMA-Factory 或 Axolotl。

每周学习节奏

  1. 看一个模块的核心概念，不追求细枝末节。
  2. 写一个最小可运行 Demo。
  3. 把 Demo 接入你的主项目。
  4. 记录踩坑、参数、效果对比。
  5. 每两周做一次复盘：哪些能力已经能独立完成，哪些只是照着教程跑通。
