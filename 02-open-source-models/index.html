学会选择、下载、运行和服务化开源模型。重点是使用模型，而不是训练基础模型。

必须理解的概念

  • Base 模型、Instruct 模型、Chat 模型的区别。
  • token、上下文窗口、采样参数、停止词。
  • 模型格式：Safetensors、GGUF、AWQ、GPTQ。
  • 量化对显存、速度和效果的影响。
  • Chat template 为什么会影响输出质量。

运行与服务化

  • Ollama：适合快速本地实验。
  • Transformers：适合理解模型加载与推理流程。
  • vLLM：适合部署 OpenAI-compatible API。
  • llama.cpp：适合 CPU/Mac/边缘设备上的 GGUF 模型。

模型选择建议

中文和中英混合任务优先尝试 Qwen 系列；代码任务优先尝试 Qwen Coder、DeepSeek Coder 等；本地入门从 7B/8B 或 14B 开始，等链路跑通后再追求更大模型。

不要只看榜单。你的业务测试集比通用榜单更重要，同一个模型在不同任务上的表现可能差异很大。

实践项目

搭建一个本地聊天服务：后端提供 `/chat` 接口，底层可以切换 Ollama 或 vLLM，支持流式输出。

验收标准

  • 可以通过 HTTP 调用本地开源模型。
  • 支持配置模型名、温度、最大输出长度。
  • 记录每次请求耗时和 token 使用情况。
