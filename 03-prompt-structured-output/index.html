Prompt 与结构化输出

让模型稳定完成任务，而不是靠运气给出好答案。这个阶段会直接影响后面的 RAG、工具调用和微调。

建议周期：2-3 周产出：结构化信息抽取服务

本阶段模块

Prompt结构化输出会话状态实践项目返回主页

Prompt 基础

  • 系统提示词、用户消息、少样本示例。
  • 约束模型回答范围、语气和格式。
  • 把复杂任务拆成分类、抽取、改写、生成等小任务。
  • 为失败答案设计修复提示词。

结构化输出

  • JSON Schema 与 Pydantic 校验。
  • 字段必填、枚举值、数组长度、类型检查。
  • 解析失败时的重试策略。
  • 把模型输出转成可入库、可调用工具的参数。

对话状态管理

  • 短期上下文：最近几轮消息。
  • 长期记忆：用户偏好、历史任务、摘要。
  • 上下文裁剪：避免把无关历史全部塞给模型。
  • 敏感信息：不要把不必要的私密数据发给模型。

实践项目

做一个“简历信息抽取器”：输入简历文本，输出姓名、技能、项目经历、工作年限、风险提示等 JSON 字段。

验收标准

  • JSON 解析成功率高于 95%。
  • 解析失败时会自动重试一次。
  • 输出通过 Pydantic 校验。
  • 保留原始输出和修复后的输出日志。
