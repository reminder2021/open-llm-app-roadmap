RAG 知识库问答

让模型基于你的文档回答问题。RAG 是 LLM 应用开发里最常见、也最容易做坏的一块。

建议周期：3-5 周产出：带引用的知识库问答系统

本阶段模块

RAG 流程检索优化答案生成实践项目返回主页

完整流程

  1. 文档解析：PDF、Markdown、HTML、Word。
  2. 清洗：去页眉页脚、目录、重复内容。
  3. 切分：按标题、段落、语义块切分。
  4. Embedding：把文本转成向量。
  5. 向量库：存储 chunk、metadata 和来源。
  6. 检索：根据问题召回相关片段。
  7. 重排：用 reranker 提升上下文质量。
  8. 生成：让模型只基于上下文回答。

检索优化重点

  • chunk size 和 overlap 不要盲目固定，要按文档结构调。
  • 混合检索：向量相似度加关键词检索。
  • Rerank 通常比单纯加大 top_k 更有效。
  • metadata 很重要，例如标题、页码、文件名、时间。
  • 查询改写可以改善用户问题过短或含糊的情况。

生成与防幻觉

  • 要求模型引用来源。
  • 上下文没有答案时明确说不知道。
  • 把引用片段和最终答案分开保存，便于排查问题。
  • 不要让模型自由补充文档外事实，除非产品明确允许。

实践项目

做一个“个人知识库问答系统”：支持上传 PDF/Markdown，自动索引，提问时返回答案和引用来源。

验收标准

  • 能展示引用文件名和段落。
  • 能重新索引文档。
  • 有 50 条测试问题。
  • 记录每个问题召回了哪些 chunk。
