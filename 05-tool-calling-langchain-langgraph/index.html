工具调用、LangChain 与 LangGraph

这一阶段回答你关心的 LangChain 问题：要学，但建议先理解工具调用底层机制，再用 LangChain/LangGraph 做工程编排。

先手写工具调用

  • 为工具设计名称、描述和参数 Schema。
  • 让模型输出工具名和参数。
  • 执行工具并把结果回填给模型。
  • 处理参数缺失、类型错误、工具失败。
  • 为高风险工具加入人工确认。
先手写一遍，你会真正理解 Function Calling 的循环：用户问题 → 模型选择工具 → 执行工具 → 观察结果 → 模型继续回答。

再学 LangChain

LangChain 适合提高开发效率，但不要把它当成黑盒魔法。重点学这些组件：

  • ChatModel：统一模型调用。
  • PromptTemplate：管理提示词模板。
  • OutputParser：解析结构化输出。
  • Tool：封装外部能力。
  • Retriever / VectorStore：接入 RAG。
  • Runnable：把多个步骤组合成链。

重点学习 LangGraph

复杂 Agent 更推荐使用 LangGraph，因为它强调状态、节点和边，适合构建可控流程。

  • 状态管理：保存任务进度、中间结果、用户偏好。
  • 分支逻辑：根据模型输出进入不同节点。
  • 循环与重试：失败后修正参数或请求用户补充。
  • 人工介入：高风险操作暂停等待确认。
  • 可观测性：更容易知道 Agent 卡在哪一步。

实践项目

做一个“个人工作助手”：能检索本地知识库、调用天气/日历/数据库等工具，并对写文件、发请求、删除数据等操作要求人工确认。

验收标准

  • 至少支持 4 个工具。
  • 工具参数有 Schema 校验。
  • 工具失败时能给出恢复策略。
  • 用 LangGraph 实现一个多步骤任务流。
