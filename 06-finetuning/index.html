LoRA/QLoRA 微调

微调不是让模型背知识，而是让模型更稳定地完成某类任务、输出某种格式或适应某种风格。

建议周期：4-8 周产出：一个领域 LoRA 模型

什么时候需要微调

适合微调

  • 固定输出格式。
  • 领域分类、抽取、改写。
  • 客服话术、写作风格。
  • 工具调用格式稳定化。

不适合优先微调

  • 大量业务知识问答。
  • 实时变化的信息。
  • 数据库查询。
  • 文档内容记忆。

数据准备

  • 从 500-2000 条高质量样本开始，质量比数量更重要。
  • 格式建议使用 instruction/input/output 或 chat messages。
  • 划分训练集、验证集、测试集。
  • 清理重复、错误、过长和泄露隐私的数据。
  • 保留负例和边界样本。

训练工具

  • Transformers：理解底层训练流程。
  • PEFT：LoRA/QLoRA 参数高效微调。
  • TRL：SFTTrainer 等训练组件。
  • LLaMA-Factory：适合快速上手和实验配置。
  • Axolotl：适合更工程化的训练配置。

训练后要评估、保存 adapter、必要时合并权重，并部署到推理服务中。

实践项目

选择一个明确任务，例如“客服意图分类与回复草稿生成”或“合同条款信息抽取”，基于 Qwen 7B/14B 做 LoRA 微调。

验收标准

  • 有清晰的数据集说明。
  • 训练前后在同一测试集上对比。
  • 记录超参数和显存占用。
  • 微调模型能通过 API 调用。
