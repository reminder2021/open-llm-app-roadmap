部署与工程化

把 Demo 变成可用服务：稳定、可观测、可扩展，并且对工具调用和数据访问有安全边界。

建议周期：3-6 周产出：可部署的 LLM 应用服务

本阶段模块

模型服务工程能力安全实践项目返回主页

模型服务

  • vLLM OpenAI-compatible API。
  • Ollama/llama.cpp 本地或轻量部署。
  • GPU 显存估算、batching、并发限制。
  • 量化模型的效果与速度取舍。
  • 多模型路由和 fallback。

工程能力

  • Docker 与 Compose。
  • 配置管理和环境变量。
  • 请求日志、错误日志、链路追踪。
  • 缓存：Embedding 缓存、答案缓存、检索缓存。
  • 任务队列：长文档解析、批量索引、离线评估。

安全边界

  • Prompt 注入防护：不要把工具权限完全交给模型。
  • 工具白名单：每个工具有明确权限和参数限制。
  • 敏感操作人工确认。
  • 数据隔离：不同用户、不同知识库权限分开。
  • 输出审计：保存关键操作记录。

实践项目

把 RAG + 工具调用助手部署为一个完整服务：前端页面、后端 API、向量库、模型服务分离，并提供基础日志和鉴权。
