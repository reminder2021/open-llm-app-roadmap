LLM 应用开发路线
总览 RAG 工具调用 微调 项目

基于开源模型的 LLM 应用开发学习路线

目标不是从零开发大模型，而是会选用开源模型、做 RAG、工具调用、LangChain/LangGraph 编排、LoRA/QLoRA 微调，并把应用部署成可用产品。

适合目标：LLM 应用开发工程师 周期建议：4-6 个月 主线：先底层机制，再框架编排

学习原则

先手写，再用框架

先手写模型调用、RAG、工具调用，理解输入输出、状态和错误处理。再学习 LangChain 与 LangGraph，效率会高很多。

先 RAG，再微调

多数业务知识问答不该直接微调。先用检索、重排、引用和评估解决，再判断是否需要微调风格、格式或领域任务。

每阶段都做项目

不要只看教程。每个阶段都交付一个可运行的小系统，最后合成一个完整的知识库与工具助手。

评估贯穿全程

Prompt、RAG、Agent、微调都要有测试集和指标，否则很难知道改动是真的变好，还是只是看起来更顺眼。

阶段目录

总览与学习方式

明确学习目标、工具栈、产出物和时间安排。

Python 与 Web API 基础

补齐 LLM 应用开发所需的 Python、HTTP、FastAPI、JSON 和异步基础。

开源模型使用与推理服务

学会本地运行、API 调用、模型格式、量化和 vLLM/Ollama 服务。

Prompt 与结构化输出

掌握提示词、JSON 输出、输出校验、重试和对话状态。

RAG 知识库问答

构建文档解析、切分、Embedding、向量检索、Rerank 和引用系统。

工具调用、LangChain 与 LangGraph

先手写 Function Calling，再学习 LangChain 组件和 LangGraph 可控 Agent。

LoRA/QLoRA 微调

学习数据构造、SFT、LoRA、QLoRA、训练、合并权重和部署。

评估体系

为 Prompt、RAG、工具调用和微调建立可复现评估。

部署与工程化

学习 Docker、GPU 显存估算、并发、日志、权限和安全防护。

综合项目作品集

用三个项目串起完整技能链：聊天助手、知识库、工具型业务助手。

建议每周至少保留 2 次完整实践时间：一次学习新模块，一次改造自己的项目。
